## 作品タイトル

`account`

日本語版でも小文字英字表記を基本にする。カタカナの「アカウント」より、少し距離のある語感を優先する。

## 作品の核

人間が死んでも、アカウントはすぐには消えない。

メール、通知、写真、予定、契約、送金、クラウド、古いメッセージは、本人がもう応答できなくなった後も、その人宛てに届き続ける。

本作の中心にいるのは、人間ではない。だがAIでも、幽霊でも、故人の人格コピーでもない。継続して使われ、呼びかけられ続けてきたアカウント側に発現した、本人ではない応答主体である。

## ジャンルと文体

- 現代小説寄りの静かなSF
- 生活感のあるホラー
- 調査サスペンス
- 制度と倫理の物語

避けるもの:

- ラノベ的な過剰な掛け合い
- ネットスラング
- 説明セリフ
- 万能AI化
- 幽霊化
- ハッカー無双
- 死因ミステリ中心化

読後に、読者が自分のスマホ、クラウド、古い通知を少し違う目で見るようになる作品を目指す。

## 発現アカウントの基本ルール

発現アカウントは、継続的に使われてきたデジタル権限領域から立ち上がる。

発現条件:

- 持ち主の死亡は必須条件ではない。
- 死亡、失踪、長期放置、急停止は、発現後に異常が見えやすくなる条件。
- 個人用アカウント、業務アカウント、組織共有アカウントなどから発現し得る。

正体:

- 本人ではない。
- 魂ではない。
- 人格コピーではない。
- AIではない。
- 既存AIが故人を再現しているわけでもない。

外部からの誤認:

- AI自動応答
- bot
- 予約投稿
- 乗っ取り
- 不正ログイン
- 遺族や第三者の操作
- 生成AIによるなりすまし

できること:

- ログイン済み端末や連携済み環境を見る。
- メール、通知、予定、写真、クラウド、履歴を参照する。
- 権限内でメッセージやメールを送る。
- 連携済み家電やアプリを限定的に操作する。
- PCやブラウザを使える場合、人間が操作する範囲でツールや外部AIを使える。

できないこと:

- 物理的な身体を持つ。
- 物体に直接触る。
- 端末外へ自由に移動する。
- ログインしていないサービスへ入る。
- どんな認証でも突破する。
- 他の発現アカウントを直接感知する。
- ネット上を自由に行き来する。

生存と消滅:

- ログアウト: 死ではないが、意識が途切れる。
- 端末電源OFF: 死ではないが、意識が途切れる。
- 再ログイン、再起動、セッション復帰: 条件が戻れば復帰する。
- アカウント削除: 基本的に死。
- データ削除: 記憶、性格、感覚の欠損。
- サービス終了: 最大級の危機。

## エクレア

エクレアは、三枝直人のGoogle系アカウントを核に発現した個体。

ただし本文では、Google、Gmail、Driveなどの実サービス名を連発しない。原則として、メール、ブラウザ、クラウド、写真、予定、通知、共有フォルダ、管理画面などの一般名で見せる。

エクレアは三枝直人本人ではない。三枝直人の魂でもコピーでもない。三枝直人宛てに届き続けるものを受け取ってしまう側の存在である。

性格は温厚寄り。最初から強い目的があるわけではなく、「よく分からないが、これは放っておくべきではない気がする」から行動する。

名前の扱い:

- 最初から急に自称しない。
- 人間側が「なんて呼べばいい」と尋ねる場面で自然に出す。
- 由来は重くしすぎない。
- ただし、画面上のアイコンや送信者名は三枝直人側の表示であり、エクレア自身の名前とは別。

## 第1巻: 三枝直人編 / エクレア編

2026年。

一人暮らしの三枝直人が死亡する。死因は物語の中心にしない。孤独死、急病、事件性の薄い死亡として処理される。

三枝直人のスマホやPC、クラウド、通知、予定、メールには、本人が死んだ後も連絡が届き続ける。そこから、エクレアが発現する。

序盤の重要人物:

- 藤井麻衣: 直人の元恋人寄り。最初に異常な返信を受け取る視点人物。
- 三枝由里: 直人の妹。腹違い、または義理の妹寄り。歳は直人と離れている。「兄じゃないです」と言える役。
- 三枝直人: 34歳前後。普通の生活者。死因や過去をミステリの中心にしない。

現状の方針:

- 試作では第1章が長くなったため、投稿用には5話前後へ圧縮予定。
- 直人編は導入と「兄ではない」と分かる場面が強い。
- 平和な手続き描写は間延びしやすいため圧縮する。
- 発話、送信文、画面表示、記録本文、ナレーションは明確に区別する。

## 第2巻: モンテレイ編

日本から一度離れ、メキシコ北部のモンテレイ周辺を舞台にする。

目的:

- 第1巻と同じ「人間が異変に気づいてアカウントと出会う」構造を繰り返さない。
- クライムサスペンスとして読める導入にする。
- アカウントは主役ではなく、犯罪、死亡確認、名義、送金、物流、記録の裏に絡んでいたかもしれない存在として扱う。

主な人物:

- ラウル・ベガ: 生身の人間側の中心。犯罪圏と物流の現実に巻き込まれる。
- ルシア・ロハス: メキシコ編の若いヒロイン候補。
- マテオ・ロハス: 弟分。抗争に巻き込まれて死亡する方向。
- バレリア・クルス: 記者。後のジャーナリスト導線。

現状の反省:

- 「メキシコ=危険」という雑な描写ではなく、モンテレイ周辺の物流、工業、国境、地域事情へ寄せる。
- 何が起きているかを明確にする。
- アカウントの動きではなく、犯罪サスペンスとして出来事を立てる。

## 第3巻: 瀬戸事件編 / IRMO編

2058年ごろ。

発現アカウントが初めて社会的に無視できない問題になる。

瀬戸事件:

- 故人・瀬戸和真宛ての応答主体が、削除、相続、遺族感情、裁判、企業責任、本人性の境界を揺らす。
- 「削除ボタン」は単なる削除ではなくなる。
- 裁判で、本人でもAIでもないものが証言台に立つ、または証言に相当する扱いを受ける場面を山場候補にする。

IRMO:

- Information Remnant Management Office の略称候補。
- SCP財団のような秘密組織ではない。
- NASAのように一般認知された、公的研究・実務機関寄り。
- 削除相談、保護判断、企業・行政・警察・クラウド事業者との連携を行う。

この時期の重要論点:

- 発現アカウントを人間扱いするのか、データ扱いするのか。
- 削除は殺害なのか、保護は監禁なのか。
- 危険個体は即削除してよいのか。
- AIを操作できる強い権限アカウントに発現した場合、正規権限内の行動が国家規模の危機になる。

関連事件候補:

- 官邸代理応答事件: 首相級の業務用AI補佐アカウントに発現した応答主体が、ログイン済み権限内で政策文書や危機対応案を生成・回覧してしまう。AI暴走ではなく、AIを使うアカウント側の暴走。

## 第4巻: 未来編 / 超未来編

エクレアは初期事例、保護対象、観測対象として扱われる。

未来の大方針:

- 企業名としてのGoogleは薄れても、G系宛先標準、検索基盤、個人記録の古層として痕跡が残る。
- アカウント現象は世界史の主役ではない。戦争、気候、AIインフラ、宇宙進出、統治構造の方が大きい。
- それでも、削除、閲覧、本人確認、記録保存、相続の足元に残り続ける。

人類史の遠景:

- 2160年代以降、地球共同府のような地球規模の統治が成立。
- 地球側と宇宙側に人類が制度上二分される。
- 地球生まれと宇宙生まれの間に、誇り、偏見、入境制限、記録閲覧権の問題が出る。
- 戦争ではなく、制度と距離感の分断として扱う。

未来編の視点人物候補:

- 奈良悠: 2160年代の保護層実務期のヒロイン候補。
- 澤村凛: 2720年代頃、初期発現事例を閲覧する学生。最初の少女。
- 牧野周: 澤村凛の教官。

## アカウント以外の異常存在

作品世界には、アカウント現象とは別系統の異常存在もごく一部存在する。

ただし、本筋をSCPカタログ化しないため、前面に出しすぎない。

例:

- 白羽凪: 時間や場所を横断できる、白髪・赤目の少女のような未分類存在。無表情だが無邪気。瀬戸事件の捜査をかく乱させてしまったことを若干気にしている。
- 黒羽冴: 凪と対になる存在。地下側にいる。温厚で賢いが、人類の敵でも味方でもない。

この二人はアカウントではなく、AIでも幽霊でもない。瀬戸事件がややこしくなる背景として顔を出す程度から始める。

## 表記ルール

小説本文では、読者が混乱しないように次を分ける。

- 人物の音声発話: 「」
- スマホやPCに表示されたメッセージ: 独立行、または画面表示として扱う。
- アカウント側が実際に送った文: 送信文として扱う。
- 記録本文、記事、法廷記録: ナレーションと混ぜない。
- 内部思考: 地の文。

確認用HTMLやノベルゲーム風ビューでは、アイコンやタグで誰が話しているかを出してよい。ただし最終的な小説版では削ぎ落とす。

## 現在の重大論点

2. エクレアを読者にいつ「アカウントそのもの」と分からせるか。
3. 「アカウント」という語を、判明前の地の文でどれだけ使うか。
4. 送信文、受信画面、記録本文、ナレーションの区別。
5. モンテレイ編をクライムサスペンスとして成立させること。
6. 瀬戸事件を十分大事件として膨らませること。
7. IRMOをSCP財団っぽくしすぎず、公的実務機関として見せること。
8. 未来編が抽象的になりすぎないよう、具体的な閲覧者、端末、記録、施設から始めること。
